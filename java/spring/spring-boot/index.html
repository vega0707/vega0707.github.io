# Introduction

最近一个哥们拉我维护公众号，恰好我也打算做这件事情，可以说是一拍即合了。但是这个公众号叫Feature社区且这个哥们呢，平时都是更新一些"较新"的技术内容。所以我也考虑搞些高层的建筑，同时通过这个建筑来重新巩固底层。

恰好这个前几天公众号发了一篇springboot启动流程分析,讲的是SpringApplication的run方法。作为补充，我打算冲头开始挖一挖Spring Boot。

当前Spring Boot的RELEASE版本为2.2.2,因此以这个版本为准。

# What's

既然我打算在这个博客中写有体系的内容，自然跑不开哲学三问：是什么，为什么，怎么做？

首先说Spring Boot是什么？官网上是这样介绍的(具体内容，自行翻译)：

Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can "just run".

We take an opinionated view of the Spring platform and third-party libraries so you can get started with minimum fuss. Most Spring Boot applications need very little Spring configuration.

# why

还是引用官网：

  1. Create stand-alone Spring applications
  2. Embed Tomcat, Jetty or Undertow directly (no need to deploy WAR files)
  3. Provide opinionated 'starter' dependencies to simplify your build configuration
  4. Automatically configure Spring and 3rd party libraries whenever possible
  5. Provide production-ready features such as metrics, health checks and externalized configuration
  6. Absolutely no code generation and no requirement for XML configuration

针对以上几个特点，我准备逐一深挖，深挖的同时再把最后一个问题->How一并解决

# How

# Create stand-alone Spring applications

顾名思义，创建单体的Spring应用。做为第一个点，显然是比较重要的或者说是显而易见的。

回顾我的开发生涯并不是很长，但是也知道到如今大行其道的Spring Boot之前，有Servlet，JSP，Struts1时代，SSH时代，SSM时代，时至今日还有很多公司的项目在用SSH，甚至还有一些公司停留在servlet。

所以为什么这里强调的是Spring applications？因为到现在，它已经可以不需要借助Struts2或Mybatis这样的框架就可以构建一个应用了。这对Spring来说，应该意义非凡吧。

至于如何快速构建，特地翻了一下官网，分享一下链接。很诡吊的是我似乎和官网的文案不谋而合了。它第一段就是：

It includes an introduction to Spring Boot, along with installation instructions. We then walk you through building your first Spring Boot application, discussing some core principles as we go.
